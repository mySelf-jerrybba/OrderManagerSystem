<插單模擬器>
簡化條件：
1.只有一個機台，該機台一天可運作24h
2.每筆訂單只考慮需要在該機台耗時多久時間，且僅只有這一道製程
3.預估工時必須包含預留時間
4.以日為單位做排程，加工流程可跨日但不可中斷
5.一天只有一筆訂單可以跨日到明天繼續工作
 
操作流程：
第一步：系統會根據資料庫的排程資料產生圖表
第二步：使用者根據自身經驗判斷填寫基本訂單資訊，以及排程資訊
第三步：系統更新資料庫
第四步：產生新的圖表

邏輯判斷：
採用<時間桶>邏輯設計，將每天可適用的時間劃分為一個桶
資料表分為：訂單、時間桶、排程

使用者選取區間內的時間桶總餘<預估工時=>false

使用者選取區間內的時間桶總餘=預估工時
  =>第一種：當日加工
  =>第二種：連續加工一天
  =>第三種：連續加工多天
使用者選取區間內的時間桶總餘>=預估工時
  =>第四種：當日加工
  =>第五種：區間過大
      =>區間過大（雖仍可排程)，但暫不考慮處理
